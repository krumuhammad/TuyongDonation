# ระบบบันทึกยอดบริจาคแบบเรียลไทม์

ระบบมี 3 หน้า ตามสิทธิ์การใช้งาน:

| หน้า | ไฟล์ | สิทธิ์ |
|---|---|---|
| หน้าจอแสดงผลทั่วไป | `index.html` | สาธารณะ ไม่ต้องล็อกอิน แสดงยอดรวมตัวใหญ่แบบเรียลไทม์ |
| หน้าผู้กรอกข้อมูล | `entry.html` | ต้องล็อกอิน สิทธิ์ `entry` หรือ `admin` |
| หน้าแอดมิน | `admin.html` | ต้องล็อกอิน สิทธิ์ `admin` เท่านั้น — จัดการรายการ/บัญชี/ตั้งค่า |
| หน้าล็อกอิน | `login.html` | ใช้ร่วมกัน ระบบจะพาไปหน้าที่ถูกต้องตามสิทธิ์อัตโนมัติ |

สถาปัตยกรรม: หน้าเว็บเป็น **ไฟล์ static (HTML/CSS/JS)** — โฮสต์ฟรีบน **GitHub Pages** ได้เลย
ส่วนฐานข้อมูล/ระบบล็อกอินใช้ **Firebase (Firestore + Authentication)** ซึ่งมี free tier
เพียงพอสำหรับงานบริจาคทั่วไป และอัปเดตแบบเรียลไทม์ให้อัตโนมัติ (ไม่ต้องเขียน backend เอง)

---

## 1) ตั้งค่า Firebase (ทำครั้งเดียว)

1. ไปที่ https://console.firebase.google.com → **Add project** → ตั้งชื่อโปรเจกต์
2. ในโปรเจกต์ ไปที่ **Build > Authentication > Sign-in method** → เปิดใช้ **Email/Password**
3. ไปที่ **Build > Firestore Database** → **Create database** → เลือกโหมด production
4. ไปที่ **Project settings (รูปเฟือง) > General > Your apps** → กด **</> (Web app)** → ตั้งชื่อ
   → คัดลอกค่า `firebaseConfig` ที่ได้ มาแทนที่ในไฟล์ `assets/firebase-config.js`
5. ไปที่ **Firestore Database > Rules** → คัดลอกเนื้อหาจากไฟล์ `firestore.rules` ในโปรเจกต์นี้ไปวางแทน → **Publish**

## 2) สร้างบัญชีแอดมินคนแรก

1. ไปที่ **Authentication > Users > Add user** → กรอกอีเมล/รหัสผ่านของแอดมิน → บันทึก
2. คัดลอก **UID** ของผู้ใช้ที่เพิ่งสร้าง (แสดงในตาราง Users)
3. ไปที่ **Firestore Database > Data** → สร้าง collection ชื่อ `staff`
   → สร้างเอกสาร โดยตั้งชื่อเอกสาร (Document ID) เป็น UID ที่คัดลอกมา
   → ใส่ฟิลด์:
   - `role` (string) = `admin`
   - `email` (string) = อีเมลที่ใช้สมัคร
   - `displayName` (string) = ชื่อที่ต้องการแสดง
4. เข้าสู่ระบบผ่าน `login.html` ด้วยบัญชีนี้ — ระบบจะพาไปหน้า `admin.html` อัตโนมัติ

จากนั้น **บัญชีเจ้าหน้าที่กรอกข้อมูลคนอื่น ๆ** สามารถเพิ่มได้ 2 ขั้นตอนต่อไปนี้:
- สร้างผู้ใช้ใหม่ใน Authentication > Users (เหมือนขั้นตอนที่ 1)
- นำ UID ไปกรอกในหน้า **admin.html > จัดการเจ้าหน้าที่** เพื่อกำหนดสิทธิ์ (เลือก "ผู้กรอกข้อมูล")
  (ไม่ต้องยุ่งกับ Firestore Console อีกต่อไป ทำผ่านหน้าแอดมินได้เลย)

## 3) ทดสอบก่อนใช้งานจริง

เปิดไฟล์ในโฟลเดอร์นี้ด้วย Visual Studio / VS Code → ใช้ส่วนขยาย **Live Server**
(หรือคำสั่ง `npx serve .`) เพื่อรันบนเครื่องตัวเองก่อน เนื่องจากหน้าเว็บใช้ ES Modules
(`type="module"`) จึงต้องเปิดผ่านเซิร์ฟเวอร์ ไม่สามารถดับเบิลคลิกเปิดไฟล์ตรง ๆ ได้

## 4) ขึ้น GitHub + เผยแพร่ด้วย GitHub Pages

```bash
git init
git add .
git commit -m "ระบบบันทึกยอดบริจาคแบบเรียลไทม์"
git branch -M main
git remote add origin https://github.com/<ชื่อผู้ใช้>/<ชื่อ-repo>.git
git push -u origin main
```

จากนั้นใน GitHub: ไปที่ repo → **Settings > Pages** → Source เลือก branch `main` → โฟลเดอร์ `/ (root)` → Save
รอสักครู่จะได้ลิงก์ประมาณ `https://<ชื่อผู้ใช้>.github.io/<ชื่อ-repo>/`

- แชร์ลิงก์หลัก (`.../`) ให้จอโปรเจกเตอร์ในงาน → แสดงยอดบริจาคตัวใหญ่แบบเรียลไทม์
- แชร์ลิงก์ `.../login.html` ให้เจ้าหน้าที่กรอกข้อมูลและแอดมิน

> **ข้อควรระวังเรื่องความปลอดภัย:** ค่าใน `firebase-config.js` (apiKey ฯลฯ) เป็นข้อมูลสาธารณะ
> โดยธรรมชาติของเว็บ static ได้อยู่แล้ว — ตัวที่ป้องกันจริงคือ **Firestore Rules** (ไฟล์
> `firestore.rules`) ซึ่งกำหนดว่าใครอ่าน/เขียนอะไรได้บ้าง จึงต้อง publish rules นี้เสมอ
> ก่อนเปิดใช้งานจริง

## โครงสร้างไฟล์

```
donation-board/
├── index.html          หน้าจอแสดงผลทั่วไป (public)
├── login.html           หน้าล็อกอินเจ้าหน้าที่
├── entry.html            หน้าผู้กรอกข้อมูล
├── admin.html            หน้าแอดมิน
├── firestore.rules       กฎความปลอดภัยของฐานข้อมูล
├── assets/
│   ├── style.css              ธีมสีทอง-กรมท่า ตัวเลขแบบป้ายสนามบิน
│   ├── firebase-config.js     ⚠️ ต้องแก้ไขด้วยค่าโปรเจกต์ของคุณเอง
│   └── firebase-init.js       ตัวช่วยเชื่อมต่อ Firebase (ไม่ต้องแก้)
└── README.md
```

## ฟีเจอร์เด่นของหน้าจอแสดงผล

- ตัวเลขยอดรวมแบบ "ป้ายแฟลบดิจิทัล" เรืองแสงทองเมื่อมียอดใหม่เข้ามา
- แถบความคืบหน้าเทียบกับเป้าหมาย (ตั้งค่าได้จากหน้าแอดมิน)
- แถบข่าววิ่งแสดงรายการบริจาคล่าสุดแบบต่อเนื่อง
- อัปเดตอัตโนมัติทันทีที่มีการกรอกข้อมูลใหม่ ไม่ต้องรีเฟรชหน้า
